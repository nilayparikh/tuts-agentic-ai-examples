## Important Framing

This lesson is not a new runnable feature slice.

It is a governance and maintenance lesson. The example is the operating model
itself: how you measure repeated AI failures, review manual corrections,
update stale guidance, validate the fixes, and avoid anti-patterns that make
agents drift over time.

## What This Lesson Focuses On

| Focus Area             | Why it matters                                                                               |
| ---------------------- | -------------------------------------------------------------------------------------------- |
| Governance cadence     | Context dies when nobody owns updates after architecture changes                             |
| Measurement signals    | Repeated AI failures and manual corrections show where context is thin or stale              |
| Anti-pattern detection | Giant instruction dumps, dead API references, and over-privileged agents create silent drift |
| Probability mindset    | Context engineering improves outcomes, but it does not remove the need for human oversight   |

## No Setup Required

There is nothing to run for this lesson folder.

Use the video and the course lesson assets to review the operating model:

1. Watch the published lesson for the final framing.
2. Read the course lesson metadata in `content/sdlc/ctx-sdlc/course/08-operating-model/`.
3. Compare your own repository against the measure → review → update → validate cycle.
